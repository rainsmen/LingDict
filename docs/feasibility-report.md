## 五、潜在挑战与解决方案

### ⚠️ **挑战1：助记图片质量不稳定**
**问题：** API搜索结果可能与单词语义不匹配  
**解决方案：**
- 建立人工审核机制，允许用户报告不合适图片
- 预先为高频词（5000词）人工筛选高质量图片
- 使用多个关键词搜索策略（英文单词 + 中文释义）

### ⚠️ **挑战2：发音API调用限制**
**问题：** 免费API有每日调用次数限制  
**解决方案：**
```
分层策略：
1. 本地缓存已查询单词的音频（约30KB/词）
2. 使用多个TTS提供商轮换（Google + Azure）
3. 离线模式自动切换到系统TTS
```

### ⚠️ **挑战3：APK体积控制**
**问题：** 词库+图片缓存可能导致应用体积过大  
**解决方案：**
- 词库压缩：使用SQLite VACUUM + 分词压缩（约30-50MB）
- 图片按需下载+懒加载
- App Bundle动态交付（按设备架构分发）

### ⚠️ **挑战4：GitHub Action签名配置**
**问题：** 签名密钥不能明文上传到GitHub  
**解决方案：**
```yaml
# 使用GitHub Secrets存储
- keystore文件Base64编码后存入secrets
- 密码存入secrets
- workflow中解码并签名
```
